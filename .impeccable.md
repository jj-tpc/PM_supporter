## Design Context

### Users
주니어~미드레벨 PM (1~5년차), "왕초보"도 포함. 매일 아침 앱을 열어 오늘 뭘 해야 하는지 파악하고, 팀 현황을 추적하고, AI의 도움으로 계획을 세우고 팀에 지시한다. 데스크톱(Windows 우선)에서 업무 시간 내내 열어두고 사용한다.

### Brand Personality
**차분한, 전문적인, 구조화된**

- "혼란스러운 하루지만, 이 앱 안에서는 정리된다"
- Asana처럼 전문 PM 도구로서의 신뢰감
- Slack/Discord 같은 소셜/채팅 중심 산만함은 배제
- 왕초보에게도 위압감 없이, 그러나 장난감처럼 보이지도 않는 톤

### Aesthetic Direction

**듀얼 모드 시스템:**

**일반 모드 (Light)**
- 화이트/밝은 베이스, 공기감 있는 레이아웃
- 풀 컬러 팔레트, 팀 현황/우선순위를 색상으로 구분
- Linear/Notion의 깔끔함 + Asana의 구조적 전문성

**Deep Work 모드 (Dark)**
- 극적인 전환 — 같은 앱이지만 완전히 다른 공간
- 다크 베이스, 모노톤 + 악센트 1색으로 팔레트 축소
- 비네팅 효과 (가장자리 어두워짐 → 시선 중앙 집중)
- 타이포 변화 (현재 태스크만 크게, 나머지 축소)
- 모든 마이크로 애니메이션 정지, 전환 속도 느리고 의도적
- 자동 풀스크린 + OS 방해금지 연동
- 윈도우 크롬(타이틀바/메뉴바) 숨김
- 클릭 타겟 확대 + 스크롤 제거 + 키보드 퍼스트
- 선택적 앰비언트 사운드 (빗소리, 카페, 화이트노이즈)
- 체크리스트 완료 시 작은 만족 사운드

**안티 레퍼런스:**
- Slack/Discord — 소셜/채팅 느낌, 알림 과다, 산만
- 과도한 게이미피케이션 — 성취 피드백은 "조용한 숫자"로, 축하 팡파레 아님

**테마 근거:**
- 일반 모드: PM은 업무 시간 밝은 사무실에서 사용 → 라이트
- Deep Work: 집중 시 외부 자극 차단, 터널 비전 유도 → 다크

### Design Principles

1. **정보는 계층적으로, 행동은 즉각적으로** — 한 화면에 모든 것을 보여주되, 가장 중요한 것이 시각적으로 가장 먼저 보인다. 클릭 1회로 행동할 수 있다.
2. **두 개의 밀도** — 일반 모드는 풍부한 전체 조망, Deep Work는 단 하나에 집중. 같은 데이터, 다른 밀도.
3. **AI는 조용한 비서** — AI 결과물은 항상 PM이 확인/수정할 수 있고, 과하게 자기 존재를 드러내지 않는다. 텍스트 버블이 아니라 구조화된 카드로 표시.
4. **도구이지 장난감이 아니다** — 전문성을 해치는 장식 요소 배제. 모든 시각 요소는 정보를 전달하거나 행동을 유도하는 목적이 있다.
5. **전환은 의미 있게** — Deep Work 진입/퇴장은 물리적 공간 이동처럼 느껴야 한다. 토글 하나지만 결과는 극적이다.

### Terminology

앱 고유 용어 체계 (여정×건축 메타포): 태스크→스텝, 보드→빌드, 컬럼→페이즈, 스프린트→스테이지, 마일스톤→체크포인트, WBS→로드맵, 팀원→크루, 팀 오더→가이드, 필터 뷰→뷰포인트. UI/UX 카피에 이 용어를 일관되게 사용한다. 상세: `docs/superpowers/specs/2026-06-24-pm-supporter-expansion-design.md` 섹션 11.

### Accessibility

- WCAG AA 준수 (색상 대비, 키보드 내비게이션)
- Deep Work 모드 특화:
  - `prefers-reduced-motion` 존중 — Deep Work에서도 전환 애니메이션 비활성화 옵션
  - 시각 자극 최소화가 기본 (모노톤 팔레트, 애니메이션 정지)
  - 앰비언트 사운드는 기본 OFF, opt-in
  - 큰 클릭 타겟 + 키보드 퍼스트는 접근성과 집중 모두에 기여
